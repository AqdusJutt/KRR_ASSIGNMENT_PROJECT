# Contributing to Multi-Agent Chat System

This document outlines how team members can contribute to this project.

## Git Workflow

1. **Create a feature branch**: `git checkout -b feature/your-feature-name`
2. **Make your changes**: Edit files, add new features, fix bugs
3. **Commit your changes**: `git commit -m "Description of changes"`
4. **Push to GitHub**: `git push origin feature/your-feature-name`
5. **Create a Pull Request**: Open a PR on GitHub

## Commit Message Guidelines

Use clear, descriptive commit messages:
- `feat: Add new memory retrieval method`
- `fix: Resolve vector search accuracy issue`
- `docs: Update README with deployment instructions`
- `refactor: Improve coordinator agent code structure`

## Code Style

- Follow PEP 8 for Python code
- Use TypeScript for frontend code
- Add docstrings to all functions and classes
- Include type hints in Python code

## Testing

Before submitting a PR:
1. Run all tests: `python run_tests.py`
2. Check that outputs are generated correctly
3. Test the web interface locally
4. Ensure Docker build works

## Team Contributions

Both team members should:
- Make meaningful commits showing individual contributions
- Review each other's code
- Maintain clear commit history
- Document major changes in commits
